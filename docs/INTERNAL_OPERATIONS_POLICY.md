## 0. 이 문서를 읽는 법

## 3. 가이드 온보딩 · 인증 · 심사

| 규칙 | 내용 | 상태 |
|---|---|---|
| 가입 경로 | 회원가입 시 '가이드' 역할 선택 → 즉시 대시보드 | ✅ |
| 가입/등록 비용 | **무료** (수수료는 예약 성사 시에만) | ✅ |
| 필수 수집(약관 기준) | 이메일, 비밀번호, 이름, 연락처, **신분증 사본, 계좌정보, 자격증명** | ⚠️ 약관엔 명시, 업로드 화면 없음 |
| 인증 배지(`is_verified`) | 운영팀 검증 가이드에 부여 | ⚠️ DB 필드만 존재, 수동 설정 |
| 자격증 업로드 화면 | **미구현** → 고객센터로 제출받아 수동 검토 | ⚠️ |

### ✅ 결정 완료 — 인증 배지 부여 기준 (Q9)
다음 **4개 요건을 모두 충족**할 때 `is_verified=true`를 수동 설정한다:
> ① 본인 신분증으로 **실명 확인** ② **연락처 인증** ③ 가이드 **자격/경력 증빙 1종 이상**(관광통역안내사 자격증, 사업자등록, 활동 증빙 등) ④ **프로필 사진이 본인**.

4개 중 하나라도 미충족이면 배지 미부여(보류 사유 회신). FAQ "인증 배지" 답변도 이 기준에 맞춘다.

### 운영 SOP (현재, 수동)
1. 가이드가 고객센터로 신분증·계좌·자격증명 제출.
2. 운영자 검토 → 통과 시 DB에서 해당 가이드 `is_verified=true`.
3. 보류/반려 사유는 이메일로 회신. 제출 서류는 검토 후 **안전 보관 또는 즉시 파기 기준** 적용(14장).

## 5. 예약 운영 규칙

### 5.1 예약 상태 머신 ✅
```
pending(수락 대기) ──가이드 수락──▶ confirmed(확정) ──결제완료──▶ paid(결제됨) ──투어 종료──▶ completed(완료)
      │                                  │
   가이드 거절                        24h 미결제
      ▼                                  ▼
  declined(거절)                    cancelled(자동취소)

여행자 취소(언제든) ─▶ cancelled(취소)
```

| 규칙 | 값 | 상태 |
|---|---|---|
| 예약 생성 시 초기 상태 | `pending` | ✅ |
| 일정 중복 경고 | 여행자 기존 예약과 겹치면 경고 후 진행 가능 | ✅ |
| **수락 후 결제 기한** | **24시간** (미결제 시 자동 취소) | ✅ |
| 예약 내역 숨김 | 여행자 본인 목록에서만 숨김(기록 보존) | ✅ |
| 가이드 수락/거절 위치 | 가이드 대시보드 / 예약 상세 | ✅ |

### ✅ 결정 완료 — 가이드 수락 기한 (Q10)
가이드가 예약 요청(`pending`)을 **72시간 무응답** 시 **자동 만료(declined 처리)**, 가이드 응답률 지표에 반영한다. *(자동 만료 로직 미구현 → 그 전까지 운영자가 72시간 초과 pending 건을 주기적으로 수동 정리)*

## 6. 결제 운영 규칙

| 규칙 | 값 | 상태 |
|---|---|---|
| 결제 시점 | 가이드 **수락 후(confirmed)** 체크아웃에서 결제 | ✅ |
| 결제 수단 | 토스페이먼츠(국내 카드·간편결제), PayPal(해외) | ✅ |
| 통화 | **전 결제 KRW 기준.** PayPal도 KRW로 청구하고 **구매자 통화 환전은 PayPal이 결제 시점 환율로 자동 처리** (고정환율 ÷1400 폐기, 2026-06) | ✅ |
| 검증 | 금액 일치, confirmed 상태, 결제 중복 방지, 상태 DONE 확인 | ✅ |
| 부분/분할 결제 | **미지원**(전액 결제) | ✅ |
| 직거래/무통장 | **금지** (피해 보호 불가) | ✅(약관) |
| 영수증 | 이용 내역(아카이브)에서 확인 | ✅ |

### 결정 완료 — 환율 (2026-06)
- **고정환율 ÷1400 폐기.** PayPal 주문을 KRW로 생성하고 환전은 PayPal이 결제 시점 환율로 처리(옵션 ① 채택). 더 이상 플랫폼이 환율을 관리하지 않으므로 환율 변동 손익 위험 없음.
- **전제 조건:** PayPal 가맹점 계정이 **KRW 수취를 지원**해야 함. Sandbox에서 KRW 주문이 정상 생성·캡처되는지 먼저 검증할 것. KRW가 거부되면 옵션 ②(매일 FX 갱신 + 버퍼)로 전환.

### ✅ 결정 완료 — PG 수수료 부담 (Q6)
결제대행(PG) 수수료는 **가이드 정산에서 공제**한다(현행 정책 문구와 일치). 정산 산식은 10장 참고.

## 7. 취소 · 환불 규정 ★핵심

### 7.1 여행자 취소 환불 테이블 (플랫폼 기본 정책) ✅(정책)
| 취소 시점 | 환불율 |
|---|---|
| **투어 집합 시각** 기준 **72시간 전**까지 | **100% 환불** |
| 투어 집합 시각 기준 **24~72시간 전** | **50% 환불** |
| 투어 집합 시각 기준 **24시간 이내** | **환불 불가** |
| 당일 No-show(미참석) | **환불 불가** |

- **기준 시각(✅ 결정, Q4):** "투어 시작"은 상품에 명시된 **투어 집합 시각**을 기준으로 한다(날짜 00:00 아님). 집합 시각 미명시 상품은 운영자가 가이드에게 확인 후 적용.
- 불가항력(천재지변·정부지침·중대 안전이슈): 별도 심사 후 예외 적용.
- 가이드 개별 정책은 위 기본 정책 **범위 내에서만** 우선 적용(여행자에게 더 유리한 방향만 허용 권장).

### 7.2 가이드 귀책 취소 ✅(정책)
- 가이드 사정으로 확정 예약 취소 시 → **여행자 전액(100%) 환불** + 가능 시 대체 가이드 재매칭 안내.
- 가이드 반복 취소는 제재 대상(12장).

### 7.3 환불 처리 기준 ⚠️ 수동
| 항목 | 값 | 상태 |
|---|---|---|
| 환불 수단 | 결제한 동일 수단으로 환급 | ⚠️ |
| **환불 실행 SLA(운영자)** | **취소 접수 후 24시간 이내 환불 실행** (담당: 이진영, ✅ Q2) | ⚠️ 수동 |
| 환불 계좌 반영 소요(고객 체감) | 실행 후 카드사/은행 일정에 따라 **영업일 3~7일** | ⚠️ |
| 자동 환불 | **미구현** — 취소 시 상태만 `cancelled`로 변경, 실제 환급은 운영자가 PG/PayPal 콘솔에서 수동 실행 | ⚠️ |
| **PG 수수료 환급(100% 환불 시)** | **PG 수수료까지 전액 환급** (결제 취소 처리, ✅ Q7) | ⚠️ |

> ⚠️ **중요 리스크:** 현재 시스템은 취소해도 **돈을 자동으로 돌려주지 않는다**(상태만 변경). 담당자(이진영)가 **취소 접수 후 24시간 이내** PG/PayPal에서 직접 환불을 실행해야 한다(15장 SOP).

### ❗[확정 필요] — 환불 세부 (잔여)
- **부분 환불(50%) 시 정산 연동:** 가이드 정산액·수수료 안분 방식(10장과 연결) — 권장: 환불 후 실수령(고객 결제 잔액) 기준으로 수수료·정산 재계산.

## 8. 노쇼 · 분쟁 · 불가항력 처리 SOP

## 9. 수수료 · 가이드 등급

### 9.1 등급별 수수료 ✅(정책)
| 등급 | 조건(월 확정 예약) | 수수료율 |
|---|---|---|
| 스타터 가이드 | 0~4건 | **15%** |
| 성장 가이드 | 5~14건 | **12%** |
| 프리미엄 가이드 | 15건 이상 | **10%** |

- **집계 기준(✅ 결정, Q5):** 등급 산정의 "예약 1건"은 **`completed`(투어 완료) 기준**으로 센다(노쇼·취소 과대집계 방지, 정산 기준과 일치). 매월 **마지막 날 23:59** 기준 누적 완료 건수 → **익월 1일부터** 새 등급 적용.
- 수수료 발생: 예약 성사 시에만(가입비·월비 없음).
- 환불과 실적: **전액 환불 건은 실적 제외**, 부분 환불 건은 실수령 금액 기준 일부 반영.

### ❗[확정 필요] (잔여)
- 등급 자동 산정 로직 **미구현** → 당분간 월말 수동 집계.

## 10. 정산 · 지급 규칙

### 10.1 정산 스케줄 ✅(정책) / ⚠️(기능 수동)
| 항목 | 값 |
|---|---|
| 정산 기준일 | 투어 **완료 처리일**(가이드/여행자 상호 확인 시점) |
| 정산 주기 | 주 1회 — **월요일 00:00 ~ 일요일 23:59** 완료 건을 한 묶음 |
| 실지급일 | **익주 수요일**(영업일 기준 순차 지급) |
| 공제 항목 | 플랫폼 수수료(등급별) + 결제대행(PG) 수수료 + 부가세(법령 기준) |
| 정산 계좌 변경 | 정산 정보 변경 요청 + 본인 인증 → 다음 주기부터 적용 |

**정산액 산식(기본):**
`지급액 = 결제금액 − 플랫폼 수수료(등급율) − PG 수수료 − 세금`

### 운영 메모 (전부 수동, 담당: 이진영 — ✅ Q2)
- 정산 전용 화면·출금 버튼 **미구현**. 담당자가 **매주 월요일에 전주(월~일) 완료 건을 집계**해 공제 후 **수요일 송금**하고 내역을 가이드에게 통지.
- 정산 계좌·출금 UI 없음 → 계좌는 고객센터로 접수받아 관리.

### ✅ 결정 — 정산 세부
- **최소 정산 금액: 없음**(완료 건은 다음 정산 주기에 즉시 지급) — Q11.
- **정산 보류: 분쟁/환불 진행 중인 건은 종결 시까지 보류** 후 정산 — Q12.

### ❗[확정 필요] (잔여)
- **투어 '완료 처리(`completed`)' 트리거:** 자동(투어 종료일 경과) vs 상호 확인 — 현재 전환 주체 불명확. 정산·등급·리뷰의 시작점이므로 정의·구현 필요.
- **세금 처리:** PG 수수료 실제 비율, 세금계산서·**원천징수(가이드 개인/사업자 구분)** 방식 — 세무 자문 후 확정.

## 12. 금지 행위 · 제재

### 12.1 제재 테이블 ✅(정책)
| 위반 | 조치 |
|---|---|
| 무단 노쇼/당일 일방취소 **누적 2회** | **14일 노출 제한** |

| 허위 정보(경력·자격·투어 내용) | **상품 비노출** + 계정 심사 |
| 중대 안전 이슈 / 폭언·차별 | 사안별 **영구 이용 제한** |

### 운영 메모
- 제재는 전부 **수동**(노출 제한/중지/정지 자동화 없음). 조치 시 **사유·증빙·일자**를 기록.
- 이의신청: 사유+증빙 첨부 접수 → **영업일 3일 내 1차 검토 회신**.

### ✅ 결정 — 단계별 제재 체계 (Q14)
**1차 경고 → 2차 노출 제한 → 3차 영구 정지**의 단계별 체계를 적용한다(위 12.1 표는 위반 유형별 기본 조치이며, 반복·경중에 따라 단계 상향). 위반별 **누적 카운트와 사유·증빙·일자를 기록**으로 관리한다. 중대 안전 이슈는 단계와 무관하게 즉시 영구 정지 가능.

## 15. 미구현 기능 수동 운영 SOP (오픈 전 필수)

현재 코드로 자동화되지 않아 **운영자가 직접 처리해야 정상 작동**하는 항목들. 정식 오픈 전 담당자·체크리스트 확정.

| # | 업무 | 트리거 | 담당 | 수동 처리 / 기한 | 우선순위 |
|---|---|---|---|---|---|
| 1 | **환불 실행** | 예약 `cancelled` 발생 | 이진영 | 토스/PayPal 콘솔에서 환불 규정(7장)대로 직접 환급, **접수 후 24시간 내** | ★최상 |
| 2 | **정산 지급** | 매주 월요일 | 이진영 | 전주(월~일) 완료 건 집계 → 공제 계산 → **수요일 송금** → 가이드 통지 | ★최상 |
| 3 | 가이드 인증 | 서류 제출 | 이진영 | 4요건(§3) 검토 후 `is_verified` 설정 | 상 |
| 4 | pending 만료 정리 | 72h 무응답 | 이진영 | 72시간 초과 `pending` 건 정리(자동화 전까지) | 상 |
| 5 | 예약 알림 | 신규 예약/확정 | 이진영 | 가이드/여행자에게 이메일·메시지 수동(추후 자동화) | 상 |
| 6 | 제재 집행 | 위반 적발 | 이진영 | 단계별(§12) 노출 제한/중지 + 기록 | 중 |
| 7 | 리뷰 삭제 | 신고 접수 | 이진영 | 기준(§11) 검토 후 DB에서 삭제 + 기록 | 중 |
| 8 | 이메일/투어 삭제 등 | 요청 | 이진영 | 수동 처리 | 하 |
